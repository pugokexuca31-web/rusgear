# RUSGEAR — витрина (storefront)

Макет витрины интернет-магазина бронезащиты **RUSGEAR**. Стек: **Next.js 16
(App Router) + React 19 + Tailwind CSS + TypeScript**. Витрина — потребитель
Storefront API платформы **Admik** (`../Uni_BD`); собственной БД/бизнес-логики нет.

Проектная документация: [`../CLAUDE.md`](../CLAUDE.md), [`../ТЗ-RUSGEAR.md`](../ТЗ-RUSGEAR.md).

## Запуск (Фаза 1 — макет на мок-данных)

```bash
pnpm install
pnpm dev        # http://localhost:3000
```

Без переменных окружения витрина работает на мок-данных (`lib/mock.ts`).

## Что реализовано (Фаза 1)

- Главная: hero, преимущества, хиты продаж, новинки.
- Каталог с фильтром по категориям и поиском.
- Карточка товара с вариантами (размер/цвет) и характеристиками.
- Корзина (localStorage) и макет оформления заказа (гостевой).
- Контентные страницы (О компании, доставка, оплата, контакты).
- Дизайн-система: строгий минимализм, desktop-first, фирменные цвета и шрифты.

## Отложено (Фаза 2)

Личный кабинет и регистрация, избранное на аккаунте, оплата СДЭК PAY,
Telegram/e-mail уведомления, CRM. См. `../CLAUDE.md §2`.

## Сращивание с Admik

Задать `NEXT_PUBLIC_ADMIK_API_URL` (+ `ADMIK_API_URL`, `STOREFRONT_API_KEY`) — и
функции в [`lib/api.ts`](lib/api.ts) пойдут в реальный API вместо мок-данных, без
правок компонентов. Маппинг эндпоинтов — `../CLAUDE.md §5` и `../Uni_BD/docs/21`.

### Шрифты

Заголовки используют `Archivo` как временную замену **HALVAR BREITSCHRIFT**
(коммерческий). При наличии лицензии — заменить в [`app/layout.tsx`](app/layout.tsx)
на `next/font/local` с файлами шрифта. Основной шрифт — Montserrat.
